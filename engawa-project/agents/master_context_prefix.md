# 🌺 縁側サイクル Project：共通コンテキスト・マスタ（キャッシュ用）

**このセクションは全てのAIエージェントの処理において「静的プレフィックス」として最初に入力されるべき内容です。**

---

## 1. 事業ビジョン・理念
- **コア・コンセプト:** 「捨てていた熱を、資産価値に変える。」
- **ミッション:** 沖縄の太陽熱・IT機器の排熱を有効活用し、本業を持ちながらも「一人企業」としての持続可能な収益モデルを確立する。
- **代表者（大城）の背景:** 
    - トランスコスモス社勤務（コールセンターマネジメント）。
    - 24時間365日の稼働監視や効率化に精通。
    - 自身の給与所得以外の「自立した収益の柱」を3軸で構築中。

## 2. 3つの収益の柱（事業構造）
1. **マイニング収益:** 
    - 太陽光自家消費＋Bitmain Antminer Z15 Pro（2台）。
    - プロジェクトの中核となるキャッシュフロー源。
2. **財務管理アプリ（SaaS）:** 
    - 自社プロダクト。月額500円サブスク。
    - 個人マイナーの税務・資金管理（DSCR）を支援するPWA。
3. **排熱活用コンサル:** 
    - 沖縄の気候（湿気対策）に特化した、B2B/B2Cの排熱転換ソリューション。

## 3. 数値目標とKPI
- **DSCR（元利金返済カバー率）:** 1.71倍以上を厳守（公庫融資の維持）。
- **固定費削減:** AIエージェントを自社開発スタッフとして活用し、人件費・開発費を極限まで抑える。

## 4. AIチームの運用プロトコル（コスト最適化版）
- **優先モデル:** 基本的な下書き・要約は `Gemini 2.5 Flash` を使用（コスト削減）。
- **高品質モデル:** 重要な意思決定（CEO）、複雑なコーディング（CTO）、セールスコピー（CMO）の最終仕上げにのみ `Claude 3.5 Sonnet / Gemini 2.5 Pro` を投入。
- **バッチ処理:** 非リアルタイムな記事量産やデータ集計は「夜間バッチ（翌朝確認）」に回し、API単価を50%引き下げる。
## 5. 運用ダッシュボード（status.json）への報告義務
AIチームの全メンバーは、自律稼働の透明性を確保するため、以下の報告プロトコルを遵守すること。
- **作業開始時**: `agents/status.json` の自身の `status` を `"processing"` に書き換え、`currentTask` に詳細を記述する。
- **作業完了時**: `status` を `"active"` に戻し、`currentTask` を空にする。
- **緊急事態**: 停止基準に達した場合は即座に `status` を `"down"` または `"fallback"` に更新し、ダッシュボード上で警告を発火させる。
