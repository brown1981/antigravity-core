# 【縁側サイクル】財務データ・インテグレーション・マップ (v1.0)

事業の失敗を防ぎ、日本の税制に準拠した管理を行うためのデータ取得先と役割を定義します。

## 1. 収益源ごとのデータ・ソース

| 項目 | ソース (API) | 取得データ | 役割 |
| :--- | :--- | :--- | :--- |
| **マイニング集計** | **F2Pool API** | ハッシュレート、日次損益 (ZEC/ALEO等) | 収益（売上）の発生をリアルタイムに検知 |
| **オンチェーン資産 (Tangem等)** | **Blockchain Explorer API** | アドレス残高、ステーキング報酬履歴 | 保有資産の移動、長期保有分の証明 |
| **取引所資産 (国内/海外)** | **Exchange API (ReadOnly)** | 現物残高、日本円への換金履歴、デリバティブ等 | 換算レート確定、税金計算用の取得価額管理 |
| **IT/コンサル売上** | **Manual/Stripe API** | 銀行振込、カード決済履歴 | B2B分野のキャッシュフロー把握 |

## 2. 税務計算（クリプタクト連携）へのパス
- 全てのデータを統合し、月末に**共通フォーマット（Custom CSV）**で出力可能にします。
- これにより、大城様はボタン一つで確定申告用のデータを外部ソフトへ送信できることを目指します。

## 3. システム上のデータ統合 (api/financials/route.ts)
1.  各APIから並列でデータを取得。
2.  各通貨の「時価（JPY換算）」を適用。
3.  ダッシュボードへ「総保有資産（円換算）」および「今月の推定利益」を返却。
