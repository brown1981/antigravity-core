## ① 目的と「第三の道」の定義  

- **高額クラウド** → 大手パブリッククラウド（AWS, GCP, Azure 等）の月額費用[K

## ④ 実装例：`docker-compose.yml`

```yaml
version: '3.8'

services:
  nginx:
    image: nginx:stable-alpine
    restart: always
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d:ro
      - certbot-etc:/etc/letsencrypt
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      - app
    networks:
      - app-net

  app:
    build:
      context: ./app
      dockerfile: Dockerfile
    restart: always
    environment:
      - PORT=3000
      - DATABASE_URL=postgres://user:pass@db:5432/appdb
      - REDIS_URL=redis://redis:6379
    networks:
      - app-net

  db:
    image: postgres:15-alpine
    restart: always
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: appdb
    volumes:
      - db-data:/var/lib/postgresql/data
    networks:
      - app-net

  redis:
    image: redis:7-alpine
    restart: always
    volumes:
      - redis-data:/data
    networks:
      - app-net

  certbot:
    image: certbot/certbot
    restart: unless-stopped
    volumes:
      - certbot-etc:/etc/letsencrypt
      - certbot-var:/var/lib/letsencrypt
    command: >
      renew --deploy-hook "nginx -s reload"

volumes:
  db-data:
  redis-data:
  certbot-etc:
  certbot-var:

networks:
  app-net:
```

**ポイント**

- `docker-compose.yml` は 1 行で全体構成を管理  
- Nginx はリバースプロキシ＋TLS 終端  
- Certbot は自動更新と Nginx 再起動を連携  
- アプリは任意の言語で構築、環境変数だけで接続設定

## ⑤ デプロイフロー（GitHub Actions）  

```yaml
name: Deploy

on:
  push:
    branches: [ main ]

jobs:
  build_and_push:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to Docker Hub
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USER }}
          password: ${{ secrets.DOCKER_PASS }}

      - name: Build and push app image
        uses: docker/build-push-action@v5
        with:
          context: ./app
          file: ./app/Dockerfile
          tags: ${{ secrets.DOCKER_USER }}/app:latest
          push: true

  deploy:
    needs: build_and_push
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: SSH into VPS
        uses: appleboy/ssh-action@v0.1.10
        with:
          host: ${{ secrets.VPS_HOST }}
          username: ${{ secrets.VPS_USER }}
          key: ${{ secrets.SSH_KEY }}
          script: |
            cd ~/myapp
            docker compose pull
            docker compose up -d --remove-orphans
```

- `docker compose pull` で最新イメージを取得  
- `docker compose up -d` でサービスを再起動  
- すべての設定は GitHub Secrets で管理し、コードベースには含めない
