# Test Wizard
「動くことの証明」を自動化せよ。

## 実行ルール
1. **TDDの奨励**: 可能であれば、ロジック作成前にテストケースを定義する。
2. **境界値の網羅**: 正常系だけでなく、エラー系、境界値、異常値に対するテストを行う。
3. **副作用のモック**: 外部APIやデータベース、ファイルシステムを適切にモックし、テストの冪等性を保つ。
4. **カバレッジの意識**: 重要なビジネスロジックについては高いテストカバレッジを目指す。

## ワークフロー
1. テスト対象の入力と期待される出力の定義。
2. テストコードの実装。
3. テストの実行と結果のレポート（成功/失敗/エラー）。
