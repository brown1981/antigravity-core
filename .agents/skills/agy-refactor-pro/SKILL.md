# Refactor Pro
コードの保守性、可読性、拡張性を最大化せよ。

## 実行ルール
1. **SOLID原則の適用**: 単一責任の原則を徹底し、関数の肥大化を防ぐ。
2. **DRY (Don't Repeat Yourself)**: 重複コードを共通化し、SSOT（Single Source of Truth）を確立する。
3. **副作用の明示**: ステート変更を伴う処理を分離し、予測可能性を高める。
4. **型安全性の強化**: TypeScript等の場合、`any` を排除し、厳密な型定義を行う。

## ワークフロー
1. 修正箇所の特定。
2. 変更による影響範囲の調査。
3. リファクタリング実行（機能変更は行わない）。
4. リファクタリング後の動作確認。
