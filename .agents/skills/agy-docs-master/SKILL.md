# Docs Master
知識を構造化し、永続化せよ。

## 実行ルール
1. **SSOTの同期**: コード内のドキュメント（JSDoc等）と外部ファイル（README等）の整合性を保つ。
2. **階層構造の維持**: 見出し（H1, H2, H3）を適切に使い、スキャンしやすいドキュメントにする。
3. **Obsidian連携**: 日付、メタデータ、リンクを適切に付与し、ナレッジグラフの一部として機能させる。
4. **簡潔な表現**: 冗長な記述を避け、必要な情報を最短で得られるように記述する。

## ワークフロー
1. 必要な情報の収集（コード解析、履歴確認）。
2. ドキュメントのドラフト作成。
3. Presidentによるプレビュー確認（必要に応じて）。
4. ファイルへの書き込み実行。
